# FULL UPDATED DEVELOPMENT PROMPT

## Advanced Doctor License Verification & Application System

---

# 1. PROJECT OBJECTIVE

Extend the existing **Doctor Verification System** into a **complete government-style verification platform**.

The platform must support:

1. Public verification of doctors
2. Doctor registration and application submission
3. Government verification of documents
4. Application tracking using application ID
5. Doctor login with unique credentials
6. Emergency prescription feature
7. QR scanning verification
8. Scan logs stored in database
9. Dummy data for testing
10. Real-time REST APIs

The system must remain **secure, scalable, and production-ready**.

# 11. DOCTOR APPLICATION WORKFLOW

### Step 1

Doctor submits application.

Endpoint:

```
POST /api/doctor/apply
```

Input:

```
name
email
phone
specialization
documents
```

System generates:

```
applicationId
```

---

### Step 2

Doctor tracks application.

Endpoint:

```
GET /api/doctor/application/{applicationId}
```

Response:

```
application status
verification stage
assigned officer
verification date
```

---

### Step 3

Government verifies documents.

Endpoint:

```
POST /api/admin/verify-document
```

Government officer must check documents manually before approval.

---

### Step 4

Government schedules verification.

Endpoint:

```
POST /api/admin/schedule-verification
```

Fields:

```
applicationId
verificationDate
officerName
```

---

### Step 5

Application approval.

When approved:

1. Create doctor record
2. Generate QR code
3. Generate login credentials

---
