## PROJECT OVERVIEW
Build a web application that allows the public to verify whether a doctor is legitimately licensed by a medical authority. The system has two interfaces:
- **Public side**: Anyone can search for a doctor by name or license number and see verification status (Verified, Expired, Revoked, or Not Found). Each verified doctor has a unique QR code that patients can scan for instant verification.
- **Admin side**: Authorized medical council personnel can log in securely to manage doctor records (add, edit, revoke, view logs).

The application must be **scalable** (stateless, ready for horizontal scaling), **secure** (authentication, authorization, input validation, encryption), and **effective** (fast searches, clear UI, error-free). ## SYSTEM ARCHITECTURE
Follow a standard layered architecture:
- **Controller Layer**: Handle HTTP requests (MVC for web pages, REST for APIs).
- **Service Layer**: Business logic, transaction management.
- **Repository Layer**: Data access via Spring Data JPA.
- **DTO Layer**: Data transfer objects to decouple API from entities.
- **Security Layer**: JWT filters, authentication manager, user details service.
- **Utility Layer**: QR code generation, validation helpers.
